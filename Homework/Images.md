# Форматы и качество изображения

## 1. Растровые изображения (Raster images)

Растровое изображение состоит из пикселей — маленьких точек.
Каждый пиксель содержит цвет. Чем больше пикселей — тем выше детализация.

Популярные растровые форматы:

### JPEG / JPG

* Использует сжатие с потерями (lossy).

* Меньший вес, но при сильном сжатии появляются артефакты.

* Подходит для: фотографий, больших изображений для веба.

### PNG

* Сжатие без потерь (lossless).

* Умеет прозрачность (alpha channel).

* Выше размер файла, чем JPEG.

* Подходит для: интерфейсов, иконок, графики, скриншотов.

### WebP

* Более современный формат от Google.

* Сжатие и с потерями, и без потерь.

* В 2–4 раза меньше по размеру чем PNG или JPEG при одинаковом качестве.

* Имеет прозрачность.

* Подходит для: всего на вебе (если нет ограничений по совместимости со старыми браузерами).

### GIF

* До 256 цветов → плохое качество.

* Есть анимации.

* Используется редко, заменён WebP/APNG.

## 2. Что влияет на качество растрового изображения?

### Разрешение (Resolution)

Количество пикселей по ширине × высоте (например, 1920×1080).

### PPI (Pixels Per Inch)

Плотность пикселей в дюйме.
Чем выше PPI, тем чётче картинка на экране или при печати.

### Сжатие

Lossy — уменьшает вес, ухудшает качество (JPEG, WebP lossy).

Lossless — сохраняет качество, но файл больше (PNG, WebP lossless).

## Векторные изображения (Vector images)

Векторная графика НЕ состоит из пикселей.
Она строится математически: точки, линии, кривые, фигуры.

### Преимущества:

* Масштабируется без потери качества (можно увеличить в 1000 раз).

* Маленький вес файла (часто).

### Недостатки:

* Плохо подходит для реалистичных фотографий.

* Сложнее для рендеринга сложных сцен.

## SVG формат (Scalable Vector Graphics)

SVG — самый популярный векторный формат в вебе.
Это текстовый файл на языке XML, где каждая фигура описана в виде кода.

Простой пример SVG формата:

<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" fill="red" />
</svg>


### Преимущества SVG:

#### 1. Масштабируемость

Никогда не теряет качество.

### **_Идеально для:_**

* логотипов,

* иконок,

* UI-графики,

* диаграмм.

#### 2. Маленький вес

Особенно в простых формах.

#### 3. Можно редактировать прямо в коде

SVG стилизуется через CSS.

Пример:

``````css
circle { fill: blue; }
``````

#### 4. Анимации

SVG можно анимировать средствами CSS и JavaScript.

#### 5. Идеален для веба

Поддерживается всеми браузерами.

### Особенности и ограничения SVG

* Не подходит для фотографий.

* Слишком сложные SVG (например, экспортированные из Figma как тысячи path-ов) → тяжёлые и медленно рендерятся.

* Бывает нужно оптимизировать (например, через svgo).

| Задача                      | Лучший формат        |
| --------------------------- | -------------------- |
| Логотип сайта               | **SVG**              |
| Иконки веб-интерфейса       | **SVG**              |
| Фото для сайта              | **WebP** / JPEG      |
| Изображения с прозрачностью | **PNG** или **WebP** |
| Анимации                    | **WebP** (animated)  |
| Скриншоты                   | PNG                  |
| Иллюстрации в Figma         | SVG                  |
